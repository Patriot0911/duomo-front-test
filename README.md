# Підхід до A/B тестування
## Мета
Метою A/B тесту є визначення, який варіант компонентів призводить до більшої зацікавленості / розуміння / тощо користувачів.

## Елемент для тестування
- **Колір основних кнопок**
  - Версія A: поточний праймрі колір (синій)
  - Версія B: альтернативний колір, який може мати асоціацію з купівлею (зелений)

## Гіпотеза
Зміна кольору кнопки вплине на ймовірність кліку користувачем. Очікується, що альтернативний колір (Версія B) підвищить показник кліків.

## Метод тестування
1. **Розподіл**
   - Відвідувачі випадково розподіляються на групу A або B
   - Кожен користувач бачить лише одну версію

2. **Відстеження**
   - Фіксуємо кліки на кнопці
   - Опційно, можна відслідковувати час перебування на сторінці, аби розуміти, як швидко відбудеться "клік"

3. **Тривалість**
   - Для статистичних показників, нам потрібно зібрати велику кількість даних
   - Наприклад, 500 користувачів на кожен варіант кнопки

4. **Аналіз**
   - Розраховуємо показник кліків для кожної версії:
     ```
     (Кількість кліків / Кількість відвідувачів) * 100%
     ```
   - Порівнюємо коеф. варіанту A та B

5. **Рішення**
   - Якщо Версія B показує значно вищий коеф взаємодіх - впроваджуємо новий колір
   - Якщо результат неочевидний - продовжуємо тестування або розглядаємо інші варіанти для випробування

## Інструменти для досліджень
- Google Optimize або власна фіча з проджекту
- Google Analytics або/і hotjar для поведінкового збережння статистики

Нам варто також внести певні деталі у процес досліджень, наприклад, за один раз - варто проводити дослідження виключно одних змін, аби статистика не збивалася на порівняння різних. Мається на увазі, саме у конкретному контексті дослідження, тобто якщо це буде відноситися до абсолютно іншої частини проєкту, іншої сторінки тощо - то це все так само має сенс, для тестування та досліджень, проте якщо це буде відноситися до, скажімо, шрифта кнопки - то це буде впливати на наше дослідження у розрізі цього тестування.
